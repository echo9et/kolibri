# Kolibri Server

Приложение Qt Quick (QML) GUI, которое обеспечивает мониторинг в реальном времени TCP соединений клиентов и данных.

## Обзор

Server — это GUI приложение, которое:
- Слушает входящие TCP соединения 
- Отображает подключенных клиентов в таблице реального времени
- Позволяет отключать клиентов
- Показывает статус устройства и сетевые метрики
- Ведет комплексные логи сервера для каждого клиента
- Обрабатывает одновременные соединения нескольких клиентов
- При нажатие на строку данных, отбразится лог клиента

## Аругменты командной строки
- `--port 12345` по умолчанию 12345

## Зависимости

- **Qt 6.5.3+**
  - `Qt::Quick` - QML/GUI фреймворк
  - `Qt::Network` - TCP/IP сетевые возможности

## Сборка

```bash
# Из корневой папки проекта
cmake -S Server -B build/Server -DCMAKE_BUILD_TYPE=Release
cmake --build build/Server --config Release
```

### Выходной файл

Собранный исполняемый файл:
- Windows: `build/Server/Release/appServer.exe`
- Linux: `build/Server/appServer`

## Использование

```bash
./appServer
```
или
```bash
./appServer --port 8080
```

Сервер будет:
1. Начнет слушать на `0.0.0.0:12345`
2. Отобразит QML-based GUI 
3. Принимать входящие соединения от клиентов
4. Обновлять UI с метриками клиентов в реальном времени
5. Вести логи всей активности сервера

## Архитектура

#### src/server/ - Сетевой сервер
- **server.h/cpp** - реализация TCP сервера
  - Организация слушания
  - Управление сокетами клиентов
  - Отправка сигналов для обновления UI
  - Потокобезопасные операции
  
#### src/core - ядро приложения
- **core.h/cpp** - используется для 'связи' логики компонентов
  - Должна быть крутая логика для работы с Qml, но ее нет
  - Нужен для будущих изминений, поэтапное и последовательное завршение работы всех компонентов
  
#### src/storage/ - Управление данными
- **storage.h/cpp** - центральное хранилище данных
  - Обработка входящих данных
  - Обновление моделей таблиц
  - Управление логами
  
- **tablemodelclients.h/cpp** - модель данных для Qml
  - Отображение списка клиентов
  - Показ статуса статуса
  - Предоставление сетевых метрик

- **modellogs.h/cpp** - модель лист для Qml
  - Логи сервера
  - Логи отдельных клиентов
  - Фильтрация логов

#### frontend/ - Пользовательский интерфейс (QML)
- **Main.qml** - основное окно приложения
  - Отображение таблицы клиентов
  - Окно выссвечлить логов
  - Управление окном

- **TableClients.qml** - отображение таблицы клиентов
  - Список всех подключенных клиентов
  - Показатели статуса
  - Нажатие для детального просмотра логов

- **Logs.qml** - интерфейс логирования
  - Общие логи сервера
  - Логи отдельных клиентов

### Модель потоков

```
┌─────────────────────────────────────────────────────┐
│   GUI Поток (QGuiApp)                               │
│  - Основное окно                                    │
│  - Построение QML                                   │
│  - Взаимодействие пользователя                      │
└────────────────────┬────────────────────────────────┘
                     │ Qt сигнал/слот
                     ↓
┌─────────────────────────────────────────────────────┐
│   Поток сервера (QThread)                           │
│  - TCP слушание                                     │
│  - Обработка соединений                             │
│  - Обработка данных                                 │
└─────────────────────────────────────────────────────┘
```

## Поток данных

```
Клиент → Сеть → Сервер (Поток) → Хранилище → Модели → QML UI
                                        ↓
                                     Логи
```
